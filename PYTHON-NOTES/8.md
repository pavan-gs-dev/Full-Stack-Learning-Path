# 8.md - Python Exception Handling

# Introduction

In real-world programs, errors can happen at any time.

Examples:

* User enters invalid input
* File does not exist
* Database connection fails
* API request fails
* Division by zero

Without exception handling, the program crashes.

Exception Handling helps us handle errors gracefully.

---

# What is an Exception?

An Exception is a runtime error that occurs while a program is running.

Example:

```python id="6u6lh5"
a = 10
b = 0

print(a / b)
```

Output:

```text id="64oz8z"
ZeroDivisionError
```

Program stops immediately.

---

# Why Exception Handling?

Without exception handling:

```text id="jmhlf3"
Program Crashes
```

With exception handling:

```text id="r88s4r"
Error Handled
Program Continues
```

---

# Common Exceptions

| Exception         | Description              |
| ----------------- | ------------------------ |
| ZeroDivisionError | Divide by zero           |
| ValueError        | Invalid value            |
| TypeError         | Wrong data type          |
| NameError         | Variable not found       |
| IndexError        | Invalid index            |
| KeyError          | Invalid dictionary key   |
| FileNotFoundError | File not found           |
| AttributeError    | Invalid object attribute |

---

# What is try?

The `try` block contains code that may cause an exception.

Syntax:

```python id="c25a1m"
try:
    risky_code
```

Example:

```python id="n7whkh"
try:

    num = 10 / 0

except:

    print("Error")
```

Output:

```text id="y6r3e7"
Error
```

---

# What is except?

The `except` block runs when an exception occurs.

Example:

```python id="8yqbh5"
try:

    print(10 / 0)

except:

    print("Cannot Divide By Zero")
```

Output:

```text id="8hxgdl"
Cannot Divide By Zero
```

---

# How try-except Works

Step 1:

```python id="ytun5g"
try:
```

Python executes code.

Step 2:

If no error:

```text id="u9ckix"
except block skipped
```

Step 3:

If error occurs:

```text id="yrm4ko"
except block executes
```

---

# Handling Specific Exceptions

Better practice:

```python id="o1nlvy"
try:

    print(10 / 0)

except ZeroDivisionError:

    print("Division By Zero Error")
```

Output:

```text id="mwb2mb"
Division By Zero Error
```

---

# Multiple Exceptions

```python id="djlwmk"
try:

    number = int(input("Enter Number: "))

    result = 100 / number

except ValueError:

    print("Invalid Number")

except ZeroDivisionError:

    print("Cannot Divide By Zero")
```

---

# Exception Object

```python id="0nq36j"
try:

    print(10 / 0)

except Exception as e:

    print(e)
```

Output:

```text id="ybrb9e"
division by zero
```

---

# Why Use Exception as e?

It gives actual error details.

Useful for:

* Debugging
* Logging
* Production systems

---

# else Block

The `else` block runs only when no exception occurs.

Example:

```python id="nzg1yz"
try:

    result = 10 / 2

except:

    print("Error")

else:

    print("Success")
```

Output:

```text id="v6x3r9"
Success
```

---

# Execution Flow

```text id="1r79l8"
try
 ↓
No Error
 ↓
else
```

---

# What is finally?

The `finally` block always executes.

Whether:

```text id="u4cy3t"
Error Occurs
or
No Error Occurs
```

---

# Example

```python id="ekx5zh"
try:

    print(10 / 0)

except:

    print("Error")

finally:

    print("Always Executes")
```

Output:

```text id="gfo9rl"
Error

Always Executes
```

---

# Why finally is Important?

Used for cleanup.

Examples:

* Closing files
* Closing database connections
* Closing network connections

---

# File Example

```python id="2m3dqf"
file = None

try:

    file = open("data.txt")

except:

    print("File Error")

finally:

    if file:

        file.close()
```

---

# Full Exception Structure

```python id="c50cza"
try:

    risky_code

except ExceptionType:

    handle_error

else:

    no_error_code

finally:

    cleanup_code
```

# Raising Exceptions

Python allows us to create exceptions manually.

Using:

```python id="g98mjs"
raise
```

---

# Example

```python id="lq0hdo"
age = -5

if age < 0:

    raise ValueError("Age Cannot Be Negative")
```

Output:

```text id="ns6ajd"
ValueError
```

---

# Why Raise Exceptions?

Used for validation.

Example:

```text id="d4v9h5"
Invalid Age

Invalid Salary

Invalid Email

Invalid Password
```

# Custom Exceptions

Python allows developers to create their own exceptions.

---

# Why Custom Exceptions?

Built-in exceptions may not describe business rules.

Example:

```text id="jyt0ol"
Bank Balance Error

Student Age Error

Salary Error
```

---

# Creating Custom Exception

```python id="s74dl4"
class InvalidAgeError(Exception):

    pass
```

---

# Using Custom Exception

```python id="wp4h4x"
class InvalidAgeError(Exception):

    pass


age = -10

if age < 0:

    raise InvalidAgeError(
        "Age Cannot Be Negative"
    )
```

Output:

```text id="m0v6b9"
InvalidAgeError
```

---

# Real Example

```python id="guk0s8"
class InsufficientBalanceError(Exception):

    pass


balance = 1000

withdraw = 5000

if withdraw > balance:

    raise InsufficientBalanceError(
        "Insufficient Balance"
    )
```

# Real-Life Example

ATM Withdrawal

```python id="kmtlv4"
try:

    amount = int(
        input("Enter Amount: ")
    )

    if amount <= 0:

        raise ValueError(
            "Invalid Amount"
        )

except ValueError as e:

    print(e)

finally:

    print("Transaction Finished")
```
