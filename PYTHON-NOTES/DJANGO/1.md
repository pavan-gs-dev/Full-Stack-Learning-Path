# 1.md - Django Introduction, Installation, Setup & Project Creation

# What is Django?

Django is a **high-level Python Web Framework** used to build secure, scalable, and powerful web applications quickly.

It follows the principle:

```text
Don't Repeat Yourself (DRY)
```

which means:

> Write code once and reuse it.

---

# What is a Web Framework?

A Web Framework provides ready-made tools to build websites and web applications.

Without a framework, you need to build everything from scratch:

* URL Routing
* Database Connection
* User Authentication
* Admin Panel
* Security
* Forms
* Sessions

A framework already provides these features.

---

# Real Life Example

Suppose you want to build an E-Commerce Website.

Without Django you need to create:

```text

Database

```

Everything manually.

With Django,

| Feature         | Django     | Flask           | FastAPI       |

# Django Architecture

Django follows the

```text
MVT Architecture
```

MVT means:

```text
Model

View

Template
```

---

# MVT Architecture

```text
Browser

   │

Request

   │

URL

   │

View

   │

Model

   │

Database

   │

View

   │

Template

   │

Response

   │

Browser
```

---

# What is Model?

Model manages database tables.

Example:

```text
Student

Employee

Product

Order
```

Every table is represented by a Model.

---

# What is View?

View contains the application logic.

Example:

```python
def home(request):

    return HttpResponse("Hello")
```

---

# What is Template?

Template is the HTML page shown to users.

Example:

```text
home.html

login.html

register.html
```

# Create Django Project

```bash
django-admin startproject myproject
```

This creates:

```text
myproject/

│
├── manage.py
│
└── myproject/
    │
    ├── settings.py
    ├── urls.py
    ├── asgi.py
    ├── wsgi.py
    └── __init__.py
```

---

# Understanding Each File

## manage.py

Used to manage the project.

Examples:

```bash
python manage.py runserver

python manage.py migrate

python manage.py createsuperuser
```

---

## settings.py

Main configuration file.

Contains:

* Installed Apps
* Database
* Security
* Templates
* Static Files

---

## urls.py

Contains URL routing.

Example:

```python
urlpatterns = [

]
```

Maps URLs to Views.

---

## wsgi.py

Used when deploying Django with WSGI servers like Gunicorn.

---

## asgi.py

Used for asynchronous applications such as WebSockets.

---

## **init**.py

Marks the folder as a Python package.

---

# Run Django Server

Move into project folder.

```bash
cd myproject
```

Run server.

```bash
python manage.py runserver
```

Output

```text
Starting development server...

http://127.0.0.1:8000/
```

Open browser:

```text
http://127.0.0.1:8000/
```

You will see:

```text
The install worked successfully!
Congratulations!
```

---

# Stop Server

Press:

```text
CTRL + C
```

# Common Django Commands

Create Project

```bash
django-admin startproject myproject
```

Run Server

```bash
python manage.py runserver
```

Run on Different Port

```bash
python manage.py runserver 9000
```

Create App

```bash
python manage.py startapp accounts
```

Make Migrations

```bash
python manage.py makemigrations
```

Apply Migrations

```bash
python manage.py migrate
```

Create Admin User

```bash
python manage.py createsuperuser
```

Open Django Shell

```bash
python manage.py shell
```

Collect Static Files

```bash
python manage.py collectstatic
```
