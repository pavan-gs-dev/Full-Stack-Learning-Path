# 7.md - Python OOP (Object-Oriented Programming)

# Introduction to OOP

OOP stands for **Object-Oriented Programming**.

It is a programming style that organizes code using **Classes** and **Objects**.

Instead of writing everything in functions and variables, OOP groups related data and behavior together.

---

# Why OOP?

Without OOP:

```python
name = "Pavan"
age = 22

print(name)
print(age)
```

For one student this is okay.

But imagine:

```text
100 Students
100 Employees
100 Products
100 Cars
```

Managing data becomes difficult.

OOP solves this problem.

---

# Real Life Example

Think about a Car.

A car has:

### Properties (Data)

```text
Brand
Color
Price
Speed
```

### Behaviors (Actions)

```text
Start()
Stop()
Accelerate()
Brake()
```

In OOP:

```text
Car = Class

My BMW = Object

My Audi = Object
```

---

# What is a Class?

A Class is a blueprint or template used to create objects.

Think:

```text
Building Plan = Class

Actual Building = Object
```

---

# Class Syntax

```python
class Student:
    pass
```

Here:

```text
Student → Class
```

---

# What is an Object?

An Object is an instance of a class.

Example:

```python
student1 = Student()
```

Now:

```text
Student → Class

student1 → Object
```

---

# First Class Example

```python
class Student:

    pass


student1 = Student()

print(student1)
```

Output:

```text
<__main__.Student object at 0x000001>
```

Python created an object in memory.

---

# Attributes

Attributes are variables inside a class.

Example:

```python
class Student:

    name = "Pavan"
    age = 22
```

Access:

```python
student = Student()

print(student.name)
print(student.age)
```

Output:

```text
Pavan
22
```

---

# Methods

Methods are functions inside a class.

Example:

```python
class Student:

    def show(self):

        print("Hello Student")
```

Call:

```python
student = Student()

student.show()
```

Output:

```text
Hello Student
```

---

# What is self?

This is one of the most important interview questions.

### self refers to the current object.

Example:

```python
class Student:

    def show(self):

        print("Hello")
```

Python automatically passes the current object.

```python
student.show()
```

Actually:

```python
Student.show(student)
```

---

# Constructor

A Constructor is a special method automatically called when an object is created.

Python Constructor:

```python
__init__()
```

---

# Why Constructor?

Used to initialize object data.

Example:

```python
class Student:

    def __init__(self):

        print("Constructor Called")
```

Create Object:

```python
student = Student()
```

Output:

```text
Constructor Called
```

---

# Constructor with Parameters

```python
class Student:

    def __init__(self, name, age):

        self.name = name
        self.age = age
```

Create Object:

```python
student = Student("Pavan", 22)
```

Access:

```python
print(student.name)
print(student.age)
```

Output:

```text
Pavan
22
```

---

# Understanding self.name

```python
self.name = name
```

Means:

```text
Object Variable = Parameter Value
```

Example:

```python
student = Student("Pavan", 22)
```

Python stores:

```text
student.name = "Pavan"

student.age = 22
```

---

# Multiple Objects

```python
class Student:

    def __init__(self, name):

        self.name = name
```

Create:

```python
s1 = Student("Pavan")

s2 = Student("Rahul")

s3 = Student("Arjun")
```

Memory:

```text
s1 → Pavan

s2 → Rahul

s3 → Arjun
```

Every object has separate data.

---

# Instance Variables

Variables belonging to objects.

Example:

```python
self.name
self.age
self.city
```

Every object gets its own copy.

---

# Class Variables

Shared by all objects.

Example:

```python
class Student:

    college = "ABC College"
```

Usage:

```python
print(Student.college)
```

All objects share the same value.

---

# Example

```python
class Student:

    college = "ABC College"

    def __init__(self, name):

        self.name = name
```

Objects:

```python
s1 = Student("Pavan")

s2 = Student("Rahul")
```

Memory:

```text
Shared:
college = ABC College

Separate:
s1.name = Pavan

s2.name = Rahul
```

# Object Methods

class Student:

# Four Pillars of OOP

The most important OOP concepts:

1. Encapsulation
2. Inheritance
3. Polymorphism
4. Abstraction

---

# Encapsulation

Wrapping data and methods together inside a class.

Example:

```python
class Student:

    def __init__(self):

        self.name = "Pavan"

    def show(self):

        print(self.name)
```

Data and methods are inside one class.

---

# Inheritance

One class can inherit properties from another class.

Example:

```python
class Animal:

    def eat(self):

        print("Eating")


class Dog(Animal):

    pass
```

Usage:

```python
dog = Dog()

dog.eat()
```

Output:

```text
Eating
```

---

# Why Inheritance?

Code Reuse.

Instead of rewriting code:

```text
Animal
   ↓
Dog
Cat
Cow
```

All can reuse Animal methods.

---

# Polymorphism

Same method name, different behavior.

Example:

```python
class Dog:

    def sound(self):

        print("Bark")


class Cat:

    def sound(self):

        print("Meow")
```

Usage:

```python
dog.sound()

cat.sound()
```

Different outputs.

---

# Abstraction

Showing only necessary details.

Example:

```text
ATM Machine
```

You only see:

```text
Withdraw
Deposit
Balance
```

You don't see internal code.

---

# Access Modifiers

Python has:

## Public

```python
self.name
```

Accessible everywhere.

---

## Protected

```python
self._name
```

Convention: internal use.

---

## Private

```python
self.__name
```

Not directly accessible.

Example:

```python
class Student:

    def __init__(self):

        self.__salary = 50000
```
