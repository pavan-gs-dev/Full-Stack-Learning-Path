# 9.md - Python Modules, Packages, pip, and Virtual Environment

# Introduction

As programs grow larger, putting all code into a single file becomes difficult.

Example:

```text
main.py
```

Contains:

* Functions
* Classes
* Database Code
* API Code
* Utility Functions

This becomes hard to manage.

Python solves this using:

```text
Modules
Packages
Libraries
Virtual Environments
```

---

# What is a Module?

A Module is simply a Python file containing Python code.

Example:

```text
calculator.py
```

```python
def add(a, b):
    return a + b

def subtract(a, b):
    return a - b
```

This file is called a module.

---

# Why Modules?

Benefits:

* Reusable Code
* Better Organization
* Easier Maintenance
* Smaller Files
* Team Collaboration

---

# Different Ways to Import

---

# Method 1

```python

```

---

# Method 2

```python

```

Output:

```text
5.0
```

# Method 3

```python
from math import sqrt, pi

print(sqrt(25))
print(pi)
```

# Method 4


# What is **name** ?

Every Python file contains:

```python
__name__
```

Example:

```python
print(__name__)
```

Output:

```text
__main__
```

---

# Why Use **name**?

To prevent code from executing when imported.

Example:

```python
def hello():
    print("Hello")


if __name__ == "__main__":

    hello()
```

# What is a Package?

A Package is a collection of modules.

Think:

```text
Folder = Package

Files = Modules
```

---

# Example Structure

```text

│
├── main.py
```

It tells Python:


# What is a Virtual Environment?

A Virtual Environment is an isolated Python environment.

Each project gets its own:

```text
Libraries
Packages
Versions
```

---

# Why Virtual Environment?

Without Virtual Environment:

```text
Project A
Requests 2.25

Project B
Requests 3.0
```

Conflict occurs.

---

# Solution

Create separate environments.

```text
Project A
Own Environment

Project B
Own Environment
```

---

# Create Virtual Environment

```bash
python -m venv myenv
```

Creates:

```text
myenv/
```

---

# Activate Virtual Environment

Windows:

```bash
myenv\Scripts\activate
```

Output:

```text
(myenv)
```

---

# Deactivate Virtual Environment

```bash
deactivate
```

---

# Install Package Inside Environment

```bash
pip install requests
```

Installed only inside:

```text
myenv
```

---

# Virtual Environment Structure

```text
myenv/

├── Scripts/
├── Include/
├── Lib/
└── pyvenv.cfg
```

An isolated Python environment for a project.


# One-Line Interview Answer

Modules help organize code into reusable Python files, packages group multiple modules together, pip installs external libraries, and virtual environments create isolated project-specific Python environments to avoid dependency conflicts.
