# Python Variables

## What is a Variable?

A variable is a name used to store a value in memory.

### Example

```python
name = "Pavan"
age = 22
```

Here:

* `name` stores `"Pavan"`
* `age` stores `22`

---

# Deep Understanding of Variables

Think of computer memory as a large warehouse.

When you write:

```python
age = 22
```

Python performs the following steps:

### Step 1: Create the Object

Python creates the integer object `22` in memory.

### Step 2: Create the Variable

Python creates the variable `age`.

### Step 3: Assign Reference

The variable `age` points to the memory location of the object `22`.

```
Variable          Object
--------          ------
age  --------->   22
```

**Important:**

Python variables do not store values directly.

They store references to objects.

---

# Variable Assignment Example

```python
a = 10
b = a
```

Memory Representation:

```
a ----\
       ---> 10
b ----/
```

Both variables point to the same object.

---

# Reassignment Example

```python
a = 10
b = a

a = 20
```

Now:

```
a -------> 20

b -------> 10
```

Python creates a new object `20` and updates the reference of `a`.

---

# Dynamic Typing in Python

Python is dynamically typed.

```python
x = 10
x = "Hello"
```

Initially:

```
x ---> 10
```

Later:

```
x ---> "Hello"
```

No type declaration is required.

## What are Binary Data Types?

Binary data types are used to store and work with raw binary data (0s and 1s).

Computers store files such as:

* Images
* Videos
* Audio Files
* PDF Documents

as binary data.

Python provides three built-in binary data types:

* `bytes`
* `bytearray`
* `memoryview`

---

# 1. Bytes

A `bytes` object stores binary data.

### Example

```python
x = b"Hello"

print(x)
```

### Output

```text
b'Hello'
```

The prefix `b` indicates that the value is stored as binary data.

---

## Why Use Bytes?

When data is transmitted over a network or stored in files, it is represented as binary values.

For example:

```text
Hello
```

Internally becomes:

```text
01001000 01100101 01101100 01101100 01101111
```

Python uses `bytes` to work with this binary representation.

---

## Bytes are Immutable

Once created, bytes cannot be modified.

### Example

```python
x = b"Hello"

x[0] = 72
```

### Output

```text
TypeError
```

---

## Features of Bytes

* Stores binary data
* Immutable
* Efficient for file and network operations

---

# 2. Bytearray

A `bytearray` is a mutable version of bytes.

### Example

```python

```

### Output

```text

```

---

## Modifying a Bytearray

```python

print(x)
```

### Output

```text
bytearray(b'Jello')
```

---

## Features of Bytearray

* Stores binary data
* Mutable
* Allows modification after creation

---

# Difference Between Bytes and Bytearray

| Bytes              | Bytearray                 |
| ------------------ | ------------------------- |
| Immutable          | Mutable                   |
| Cannot be modified | Can be modified           |
| Faster             | Slightly slower           |
| Uses less memory   | Uses slightly more memory |

---

# 3. Memoryview

A `memoryview` provides direct access to existing binary data without creating a copy.

### Example

```python
data = bytes([65, 66, 67])

view = memoryview(data)

print(view[0])
```

### Output

```text
65
```

The value `65` represents the ASCII code for the character `A`.

---

# Why Use Memoryview?

Consider a large file:

```text
10 GB Video File
```

Without memoryview:

```python
new_data = old_data.copy()
```

Python creates a new copy of the data, consuming additional memory.

With memoryview:

```python
view = memoryview(old_data)
```

No copy is created.

Python accesses the original data directly.

---

## Benefits of Memoryview

* Faster processing
* Lower memory usage
* Better performance for large files

---

# Real-World Analogy

## Normal Copy

```text
Original Book
      ↓
Create Another Book
```

More memory is used.

---

## Memoryview

```text
Original Book
      ↓
Read Through a Window
```

No additional copy is created.

The same data is accessed directly.
