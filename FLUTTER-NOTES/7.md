# 📘 Dart OOP (Object-Oriented Programming)

<h1 align="center">Classes, Objects, Constructors & Encapsulation</h1>

Object-Oriented Programming (OOP) is one of the most important concepts in Dart and Flutter.

Flutter itself is built using OOP.

Everything you create in Flutter is based on:

* Classes
* Objects
* Constructors
* Encapsulation
* Inheritance
* Polymorphism
* Abstraction

---

# What is OOP?

OOP stands for:

```text
Object-Oriented Programming
```

It is a programming style that organizes code into objects.

Instead of storing everything in variables and functions, we create objects that contain:

* Data (Properties)
* Actions (Methods)

---

# Real Life Example

Think about a Car.

A Car has:

Properties:

```text
Color
Brand
Speed
Model
```

Actions:

```text
Start()
Stop()
Drive()
Brake()
```

In OOP:

```text
Car = Class

BMW = Object
Audi = Object
Tesla = Object
```

---

# 1. Class

## What is a Class?

A Class is a blueprint or template.

A class defines:

* Variables
* Functions
* Behavior

A class itself does not occupy memory.

It only acts as a design.

---

## Real Life Example

Blueprint of a house.

Before building a house, engineers create a design.

That design is like a Class.

The actual house is like an Object.

---

# Creating a Class

```dart
class Student {

}
```

This is the simplest class.

---

# Class with Variables

```dart
class Student {

  String name = "Pavan";
  int age = 22;

}
```

Here:

```text
name
age
```

are properties (variables).

---

# Class with Methods

```dart
class Student {

  String name = "Pavan";

  void study() {
    print("Studying Dart");
  }

}
```

study() is a method.

---

# 2. Object

## What is an Object?

An Object is a real instance of a class.

Class = Design

Object = Real thing

---

## Example

```dart
class Student {

  String name = "Pavan";

}
```

Creating Object:

```dart
void main() {

  Student student1 = Student();

}
```

Here:

```text
Student
```

is Class.

```text
student1
```

is Object.

---

# Accessing Properties

```dart
class Student {

  String name = "Pavan";
  int age = 22;

}

void main() {

  Student student1 = Student();

  print(student1.name);
  print(student1.age);

}
```

Output:

```text
Pavan
22
```

---

# Multiple Objects

```dart
class Student {

  String name = "";
  int age = 0;

}
```

```dart
void main() {

  Student s1 = Student();
  Student s2 = Student();

  s1.name = "Pavan";
  s2.name = "Rahul";

  print(s1.name);
  print(s2.name);

}
```

Output:

```text
Pavan
Rahul
```

Each object has its own data.

---

# Why Classes and Objects?

Without OOP:

```dart
String name1 = "Pavan";
String name2 = "Rahul";
String name3 = "Kiran";
```

This becomes difficult to manage.

With OOP:

```dart
Student s1 = Student();
Student s2 = Student();
Student s3 = Student();
```

Much cleaner.

---

# 3. Constructor

## What is a Constructor?

A Constructor is a special method used to initialize an object.

It runs automatically when an object is created.

---

## Example

```dart
class Student {

  Student() {
    print("Constructor Called");
  }

}

void main() {

  Student s1 = Student();

}
```

Output:

```text
Constructor Called
```

---

# Why Constructor?

Imagine creating a student.

Every student needs:

```text
Name
Age
```

Instead of assigning values later, we can provide them during object creation.

---

# Parameterized Constructor

```dart
class Student {

  String name;
  int age;

  Student(this.name, this.age);

}
```

Creating Object:

```dart
void main() {

  Student s1 = Student(
    "Pavan",
    22
  );

  print(s1.name);
  print(s1.age);

}
```

Output:

```text
Pavan
22
```

---

# Understanding this Keyword

```dart
Student(this.name, this.age);
```

means:

```dart
this.name = name;
this.age = age;
```

The keyword:

```dart
this
```

refers to the current object.

---

# Long Version

```dart
class Student {

  String name;
  int age;

  Student(String name, int age) {

    this.name = name;
    this.age = age;

  }

}
```

---

# Short Version

```dart
Student(this.name, this.age);
```

Most Dart developers use the short version.

# 4. Encapsulation

## What is Encapsulation?

Encapsulation means:

```text
Hide Data
+
Control Access
```

We should not allow direct access to important data.

Instead we provide controlled access.

---

# Real Life Example

ATM Machine

You cannot directly touch:

```text
Bank Database
Account Balance
```

You interact through:

```text
Withdraw
Deposit
Check Balance
```

This is Encapsulation.

---

# Private Variable

In Dart:

```dart
_variableName
```

is private.

---

## Example

```dart
class BankAccount {

  double _balance = 10000;

}
```

The underscore:

```text
_
```

makes the variable private.

---

# Getter

Getter is used to read data.

```dart
class BankAccount {

  double _balance = 10000;

  double getBalance() {
    return _balance;
  }

}
```

---

# Using Getter

```dart
void main() {

  BankAccount account =
      BankAccount();

  print(
    account.getBalance()
  );

}
```

Output:

```text
10000
```

---

# Setter

Setter is used to update data safely.

```dart
class BankAccount {

  double _balance = 0;

  void setBalance(
      double amount) {

    if(amount > 0) {
      _balance = amount;
    }

  }

}
```

---

# Complete Encapsulation Example

```dart
class BankAccount {

  double _balance = 0;

  void deposit(double amount) {

    if(amount > 0) {
      _balance += amount;
    }

  }

  double getBalance() {
    return _balance;
  }

}

void main() {

  BankAccount account =
      BankAccount();

  account.deposit(5000);

  print(
    account.getBalance()
  );

}
```

Output:

```text
5000
```

---

# Why Encapsulation?

Without Encapsulation:

```dart
account.balance = -10000;
```

Invalid value.

---

With Encapsulation:

```dart
if(amount > 0)
```

Invalid values are blocked.

Data remains safe.
