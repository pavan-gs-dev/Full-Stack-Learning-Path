# 📘 Exception Handling in Dart

<h1 align="center">Exception Handling in Dart</h1>

Exception Handling is used to handle errors that occur while a program is running.

Without exception handling, an application may crash when an error occurs.

Exception handling allows us to:

* Prevent application crashes
* Show user-friendly error messages
* Handle unexpected situations safely
* Write professional applications

---

# What is an Exception?

An Exception is an error that occurs during program execution.

Example:

```dart
int result = 10 ~/ 0;
```

Output:

```text
Unhandled Exception:
IntegerDivisionByZeroException
```

The program crashes because dividing by zero is not allowed.

---

# Why Do We Need Exception Handling?

Imagine a Flutter application:

* User enters invalid data
* API server is down
* Internet connection is lost
* File is missing
* Database connection fails

If errors are not handled properly, the application may crash.

Exception handling helps us manage these situations safely.

---

# Types of Errors

## 1. Compile-Time Error

Errors found before the program runs.

Example:

```dart
print("Hello"
```

Output:

```text
Syntax Error
```

The code will not compile.

---

## 2. Runtime Error

Errors that occur while the program is running.

Example:

```dart
int result = 10 ~/ 0;
```

Output:

```text
IntegerDivisionByZeroException
```

Program crashes during execution.

---

# try and catch

The most common way to handle exceptions.

## Syntax

```dart
try {
  // risky code
}
catch (e) {
  // error handling
}
```

---

# Example

```dart
void main() {

  try {

    int result = 10 ~/ 0;

    print(result);

  } catch (e) {

    print("Error Occurred");

  }

}
```

Output:

```text
Error Occurred
```

Instead of crashing, the error is handled.

---

# Understanding try Block

The try block contains code that may cause an exception.

Example:

```dart
try {

  int result = 10 ~/ 0;

}
```

This code is risky because division by zero may occur.

---

# Understanding catch Block

The catch block runs only when an exception occurs.

Example:

```dart
catch (e) {

  print("Something went wrong");

}
```

---

# Accessing Error Details

The variable:

```dart

```

contains information about the error.

Example:

```dart
void main() {

  } catch (e) {

    print(e);

  }

}
```

Output:

```text

```

Now we know exactly what went wrong.

# on Keyword

The on keyword handles a specific exception type.

## Syntax

```dart
try {

}
on ExceptionType {

}
```

---

# Example

```dart
void main() {

  try {

    int result = 10 ~/ 0;

  }
  on IntegerDivisionByZeroException {

    print("Cannot Divide By Zero");

  }

}
```

Output:

```text
Cannot Divide By Zero
```

---

# Why Use on?

Using catch:

```dart
catch(e)
```

handles every error.

Using on:

```dart
on IntegerDivisionByZeroException
```

handles only that specific error.

More precise and professional.

---

# on + catch Together

You can use both.

Example:

```dart
void main() {

  try {

    int result = 10 ~/ 0;

  }

  on IntegerDivisionByZeroException catch(e) {

    print(e);

  }

}
```

Output:

```text
IntegerDivisionByZeroException
```

---

# finally Block

The finally block always executes.

Even if an exception occurs.

Even if no exception occurs.

---

# Syntax

```dart
try {

}
catch(e) {

}
finally {

}
```

---

# Example

```dart
void main() {

  try {

    int result = 10 ~/ 0;

  }

  catch(e) {

    print("Error");

  }

  finally {

    print("Finally Executed");

  }

}
```

Output:

```text
Error
Finally Executed
```

---

# Why Use finally?

Used for cleanup operations.

Examples:

* Close database connection
* Close files
* Logout user
* Release resources

---

# Example Without Error

```dart
void main() {

  try {

    print("Success");

  }

  catch(e) {

    print("Error");

  }

  finally {

    print("Always Runs");

  }

}
```

Output:

```text
Success
Always Runs
```

# throw Keyword

Used to create exceptions manually.

Sometimes we want to generate our own errors.

---

Output:


# Custom Exceptions

Large applications often create their own exceptions.

---

# Creating Custom Exception

```dart
class InvalidAgeException
    implements Exception {

  String message;

  InvalidAgeException(
      this.message);

}
```

---

# Using Custom Exception

```dart
class InvalidAgeException
    implements Exception {

  String message;

  InvalidAgeException(
      this.message);

}

void main() {

  try {

    int age = 15;

    if(age < 18) {

      throw InvalidAgeException(
          "Age is too low");

    }

  }

  catch(e) {

    print(e);

  }

}
```

---

# Better Custom Exception

```dart
class InvalidAgeException
    implements Exception {

  String message;

  InvalidAgeException(
      this.message);

  @override
  String toString() {

    return message;

  }

}
```

Output:

```text
Age is too low
```

Cleaner output.

# Best Practices

✅ Use try-catch for risky code

✅ Use specific exceptions when possible

✅ Use finally for cleanup

✅ Create custom exceptions for business logic

❌ Do not leave exceptions unhandled

❌ Do not use try-catch everywhere unnecessarily
