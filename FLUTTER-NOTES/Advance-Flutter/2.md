# 1. const Keyword

Example:

```dart
const MyApp()
```

Const means:

```text
Compile Time Constant
```

The object is created once and reused.

Without const:

```dart
MyApp()
```

Flutter creates a new object every time.

With const:

```dart
const MyApp()
```

Flutter can reuse the same object.

Benefits:

* Better performance
* Less memory usage
* Faster rebuilds

Example:

```dart
const Text("Hello")
```

Good Practice:

```dart
const Icon(Icons.home)
const SizedBox(height: 10)
const Text("Welcome")
```

---

# 2. What is super.key ?

Code:

```dart
const MyApp({super.key});
```

This is constructor syntax.

Old Way:

```dart
const MyApp({Key? key}) : super(key: key);
```

Modern Flutter:

```dart
const MyApp({super.key});
```

Both are same.

---

## What is Key?

A Key helps Flutter identify widgets uniquely.

Example:

```dart
Text("Hello")
```

Flutter creates widget.

When rebuild happens:

Flutter asks:

```text
Is this same widget?
Or a new widget?
```

Keys help Flutter identify widgets.

Example:

```dart
Text(
  "Hello",
  key: ValueKey("txt1"),
)
```

---

## Why pass key to super?

Because StatelessWidget already has a constructor:

```dart
StatelessWidget({
  Key? key
})
```

We forward our key to parent class.

```dart
super.key
```

means

```dart
Pass key to parent widget.
```

---

# Constructor Breakdown

```dart
const MyApp({super.key});
```

Meaning:

```text
const → Constructor is constant

MyApp → Constructor Name

{ } → Named Parameters

super.key → Pass key to parent
```

# 3. required Keyword

Code:

```dart
required this.title
```

Required means:

```text
You MUST provide value.
```

Example:

```dart
class User {
  final String name;

  User({
    required this.name,
  });
}
```

Correct:

```dart
User(name: "Pavan");
```

Wrong:

```dart
User();
```

Error:

```text
Missing required argument
```

# 4. this.title

Code:

Equivalent to:

```dart

```

Flutter automatically assigns.

# 6. @override

Code:

```dart
@Override
Widget build(BuildContext context)
```

Actually:

```dart
@override
```

(not @Override like Java)

---

## What is Override?

Parent class already contains method.

Child class replaces it.

Example:

```dart
class Animal {

  void sound() {
    print("Animal");
  }
}
```

Child:

```dart
class Dog extends Animal {

  @override
  void sound() {
    print("Bark");
  }
}
```

Now Dog uses its own version.

---

## Flutter Example

StatelessWidget already has:

```dart
Widget build(BuildContext context)
```

We provide our implementation.

```dart
@override
Widget build(BuildContext context) {
  return Container();
}
```

---

# Why use @override?

Benefits:

* Compiler checking
* Prevent mistakes
* Better readability

Wrong:

```dart
Widget bulid(BuildContext context)
```

Notice typo:

```text
bulid
```

Compiler catches it.

# 11. setState()

Code:

```dart

  _counter++;
});
```

Most Important Concept.

Without setState:

```dart

```

Value changes.

---

```dart

});
```

Flutter rebuilds screen.

# Interview Questions

### Why use const?

Reduces rebuilds and improves performance.

### Why use super.key?

Passes widget key to parent class.

### What does required do?

Makes parameter mandatory.

### Difference between Widget and State?

Widget = Configuration

State = Mutable Data

### Why use setState()?

Notifies Flutter to rebuild UI.

### What does widget.title mean?

Access parent widget properties inside State class.

### What does @override do?

Replaces parent class method implementation.
