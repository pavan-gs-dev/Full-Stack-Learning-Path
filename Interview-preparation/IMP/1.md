# FastAPI Interview Questions

### Q1: What is FastAPI?

**Answer:**
FastAPI is a modern Python framework used to build high-performance REST APIs.

---

### Q2: Why is FastAPI popular?

**Answer:**

* Fast performance
* Easy to learn
* Automatic API documentation
* Type validation

---

### Q3: What are the advantages of FastAPI?

**Answer:**

* High speed
* Automatic Swagger UI
* Async support
* Easy development

---

### Q4: What is Swagger UI?

**Answer:**
Swagger UI is an automatically generated page used to test APIs.

Default URL:

```text
/docs
```

---

### Q5: What is Pydantic?

**Answer:**
Pydantic is used for request validation and data validation in FastAPI.

---

### Q6: What is dependency injection in FastAPI?

**Answer:**
A way to reuse common functionality like database connections and authentication.

---

### Q7: How does FastAPI connect to PostgreSQL?

**Answer:**
Using:

* SQLAlchemy
* psycopg2
* asyncpg

---

### Q8: Why did you choose FastAPI?

**Answer:**
Because it is fast, lightweight, and easy for building APIs.

---

# Django Interview Questions

### Q9: What is Django?

**Answer:**
Django is a high-level Python web framework used to build web applications quickly.

---

### Q10: What is Django famous for?

**Answer:**
"Battery Included Framework"

Because many features are built-in.

---

### Q11: Features of Django?

**Answer:**

* Authentication
* Admin Panel
* ORM
* Security
* URL Routing

---

### Q12: What is Django ORM?

**Answer:**
ORM allows interaction with databases using Python code instead of SQL.

Example:

```python
User.objects.all()
```

---

### Q13: What is migration?

**Answer:**
Migration is used to create and update database tables.

Commands:

```bash
python manage.py makemigrations

python manage.py migrate
```

---

### Q14: What is Django Admin?

**Answer:**
A built-in dashboard used to manage application data.

---

### Q15: Difference between Django and FastAPI?

**Answer:**

Django:

* Full framework
* Built-in features

FastAPI:

* API framework
* Faster performance

---

### Q16: What is MTV architecture?

**Answer:**

Model
Template
View

Used in Django.

# JWT Authentication

### Q32: What is Authentication?

**Answer:**
Authentication verifies the identity of a user.

---

### Q33: What is Authorization?

**Answer:**
Authorization determines what a user can access.

---

### Q34: What is JWT?

**Answer:**
JWT (JSON Web Token) is used to securely transfer user information.

---

### Q35: Why use JWT?

**Answer:**

* Secure
* Stateless
* Easy authentication

---

### Q36: What are the parts of JWT?

**Answer:**

1. Header
2. Payload
3. Signature

---

### Q37: What happens after login?

**Answer:**
Server generates JWT and sends it to the client.

Client sends JWT with future requests.

---

### Q38: Difference between Session and JWT?

**Answer:**

Session:

* Stored on server

JWT:

* Stored on client

---

### Q39: Which is more scalable?

**Answer:**
JWT because the server doesn't need to store session data.

---

# SPA Architecture Interview Questions

### Q53: What is SPA?

**Answer:**
SPA stands for Single Page Application.

The page loads once and updates content dynamically without reloading.

---

### Q54: Why use SPA?

**Answer:**

* Faster user experience
* App-like navigation
* Less page reloads

---

### Q55: How does SPA work?

**Answer:**
JavaScript loads content dynamically and updates the page.

---

### Q56: What is Client-Side Routing?

**Answer:**
Routing handled by JavaScript in the browser instead of the server.

---

### Q57: What is the biggest advantage of SPA?

**Answer:**
Fast navigation without full page reload.

---

### Q58: What is the biggest disadvantage of SPA?

**Answer:**
SEO can be difficult without SSR.

---

### Q59: What is SSR?

**Answer:**
SSR (Server Side Rendering) renders HTML on the server before sending it to the browser.

---

### Q60: Explain your SPA project.

**Answer:**
I built a Single Page Application using PHP and Vanilla JavaScript. Navigation was handled dynamically without page reloads, improving user experience and performance. The application included authentication, dashboards, admin panels, and API integration.
