# PostgreSQL Interview Questions and Answers for Freshers

---

# PostgreSQL Basics

### Q1: What is PostgreSQL?

**Answer:**
PostgreSQL is an open-source relational database management system (RDBMS) used to store, manage, and retrieve data.

---

### Q2: Why is PostgreSQL used?

**Answer:**

* Data storage
* High performance
* Reliability
* ACID compliance
* Supports complex queries

---

### Q3: Difference between PostgreSQL and MySQL?

**Answer:**

PostgreSQL:

* More advanced features
* Better support for complex queries
* Supports JSON and advanced indexing

MySQL:

* Simpler
* Faster for small applications
* Easier to learn

---

### Q4: What is a Database?

**Answer:**
A database is an organized collection of related data.

Example:

* User data
* Product data
* Orders data

---

### Q5: What is a Table?

**Answer:**
A table stores data in rows and columns.

Example:

| id | name  | age |
| -- | ----- | --- |
| 1  | Pavan | 22  |

---

### Q6: What is a Row?

**Answer:**
A row represents a single record in a table.

---

### Q7: What is a Column?

**Answer:**
A column represents an attribute of the data.

Example:

* id
* name
* email

---

# Keys

### Q8: What is a Primary Key?

**Answer:**
A Primary Key uniquely identifies each row in a table.

Example:

```sql
id SERIAL PRIMARY KEY
```

---

### Q9: Can a Primary Key contain NULL values?

**Answer:**
No.

---

### Q10: What is a Foreign Key?

**Answer:**
A Foreign Key creates a relationship between two tables.

Example:

```sql
user_id INTEGER REFERENCES users(id)
```

---

### Q11: Why use Foreign Keys?

**Answer:**
To maintain data integrity between tables.

---

### Q12: Difference between Primary Key and Foreign Key?

**Answer:**

Primary Key:

* Uniquely identifies a row.

Foreign Key:

* References another table.

---

# Constraints

### Q13: What are Constraints?

**Answer:**
Rules applied to table columns.

---

### Q14: Common Constraints?

**Answer:**

* PRIMARY KEY
* FOREIGN KEY
* UNIQUE
* NOT NULL
* CHECK
* DEFAULT

---

### Q15: What is UNIQUE Constraint?

**Answer:**
Prevents duplicate values.

Example:

```sql
email VARCHAR(255) UNIQUE
```

---

### Q16: What is NOT NULL?

**Answer:**
Ensures a column cannot contain NULL values.

---

### Q17: What is DEFAULT Constraint?

**Answer:**
Provides a default value when none is specified.

Example:

```sql
status VARCHAR(20) DEFAULT 'active'
```

---

# SQL Commands

### Q18: What is DDL?

**Answer:**
Data Definition Language.

Commands:

* CREATE
* ALTER
* DROP
* TRUNCATE

---

### Q19: What is DML?

**Answer:**
Data Manipulation Language.

Commands:

* INSERT
* UPDATE
* DELETE

---

### Q20: What is DQL?

**Answer:**
Data Query Language.

Command:

* SELECT

---

### Q21: What is DCL?

**Answer:**
Data Control Language.

Commands:

* GRANT
* REVOKE

---

# CRUD Operations

### Q22: How do you insert data?

**Answer:**

```sql
INSERT INTO users(name,email)
VALUES('Pavan','pavan@gmail.com');
```

---

### Q23: How do you retrieve data?

**Answer:**

```sql
SELECT * FROM users;
```

---

### Q24: How do you update data?

**Answer:**

```sql
UPDATE users
SET name='Kumar'
WHERE id=1;
```

---

### Q25: How do you delete data?

**Answer:**

```sql
DELETE FROM users
WHERE id=1;
```

---

# WHERE Clause

### Q26: What is WHERE clause?

**Answer:**
Used to filter records.

Example:

```sql
SELECT * FROM users
WHERE age > 18;
```

---

### Q27: Difference between WHERE and HAVING?

**Answer:**

WHERE:

* Filters rows before grouping.

HAVING:

* Filters grouped data after aggregation.

---

# Joins (Very Important)

### Q28: What is a JOIN?

**Answer:**
JOIN combines data from multiple tables.

---

### Q29: What is INNER JOIN?

**Answer:**
Returns matching records from both tables.

---

### Q30: What is LEFT JOIN?

**Answer:**
Returns all rows from left table and matching rows from right table.

---

### Q31: What is RIGHT JOIN?

**Answer:**
Returns all rows from right table and matching rows from left table.

---

### Q32: What is FULL JOIN?

**Answer:**
Returns all rows from both tables.

---

### Q33: Which JOIN is most commonly used?

**Answer:**
INNER JOIN.

---

# Indexing (Very Important)

### Q34: What is an Index?

**Answer:**
An index improves query performance by allowing faster data retrieval.

---

### Q35: Why use Indexes?

**Answer:**

* Faster searches
* Faster filtering
* Better query performance

---

### Q36: Disadvantage of Indexes?

**Answer:**

* Uses extra storage
* Slightly slower INSERT and UPDATE operations

---

### Q37: Which columns should be indexed?

**Answer:**

* Frequently searched columns
* Foreign keys
* WHERE clause columns

---

# Normalization

### Q38: What is Normalization?

**Answer:**
Normalization reduces data redundancy and improves consistency.

---

### Q39: Why use Normalization?

**Answer:**

* Avoid duplicate data
* Improve database design
* Maintain consistency

---

# Transactions

### Q40: What is a Transaction?

**Answer:**
A transaction is a group of SQL operations executed as a single unit.

---

### Q41: Transaction Commands?

**Answer:**

```sql
BEGIN;
COMMIT;
ROLLBACK;
```

---

### Q42: What is COMMIT?

**Answer:**
Permanently saves changes.

---

### Q43: What is ROLLBACK?

**Answer:**
Undoes changes if an error occurs.

---

# ACID Properties

### Q44: What does ACID stand for?

**Answer:**

A - Atomicity
C - Consistency
I - Isolation
D - Durability

---

### Q45: Why are ACID properties important?

**Answer:**
They ensure reliable and consistent transactions.

---

**Answer:**

Examples:

* SQLAlchemy
* Django ORM

---

### Q52: Difference between DELETE, TRUNCATE, and DROP?

**Answer:**

DELETE:

* Removes selected rows.

TRUNCATE:

* Removes all rows quickly.

DROP:

* Removes the entire table.

---
