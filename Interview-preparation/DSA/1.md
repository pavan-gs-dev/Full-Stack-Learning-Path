# 1. Arrays

## Definition
An Array is a collection of elements stored in contiguous memory locations.

Example:

[10, 20, 30, 40, 50]

Index:
0 1 2 3 4

Access:
arr[2] = 30

## Time Complexity

Access = O(1)
Search = O(n)
Insert = O(n)
Delete = O(n)

## Interview Questions

Q1: What is an Array?
Answer:
An Array is a linear data structure that stores elements in contiguous memory locations.

Q2: Why is array access O(1)?
Answer:
Because we can directly calculate memory location using index.

Q3: Difference between Array and Linked List?

Array:
- Fixed size
- Fast access

Linked List:
- Dynamic size
- Slow access

## Example

Python:

arr = [10,20,30,40]

print(arr[1])

Output:
20

# 2. Linked List

## Definition

A Linked List is a collection of nodes.

Each node contains:
- Data
- Next Pointer

Example:

10 -> 20 -> 30 -> NULL

## Types

1. Singly Linked List
2. Doubly Linked List
3. Circular Linked List

## Time Complexity

Insert at beginning = O(1)
Delete at beginning = O(1)
Search = O(n)

## Interview Questions

Q1: Why use Linked List?

Answer:
When size changes frequently.

Q2: Difference between Array and Linked List?

Array uses contiguous memory.
Linked List uses non-contiguous memory.

Q3: What is a Node?

Answer:
A node contains data and address of next node.

## Example

class Node:
    def __init__(self,data):
        self.data = data
        self.next = None

# 9. Searching

## Linear Search

Check elements one by one.

Time:
O(n)

Example:

[10,20,30,40]

Find 30

Check:
10
20
30

Found

## Binary Search

Works only on sorted arrays.

Example:

[10,20,30,40,50]

Middle = 30

Found immediately.

Time:
O(log n)

## Interview Questions

Q1: When can Binary Search be used?

Answer:
Only on sorted data.

Q2: Linear vs Binary Search?

Linear = O(n)

Binary = O(log n)

# 10. Dynamic Programming (DP)

## Definition

DP solves problems by storing previously calculated results.

Main Idea:

Don't calculate same thing again and again.

## Example

Fibonacci

Bad:

fib(5)
calls fib(4)
calls fib(3)
again calls fib(3)

Repeated calculations.

DP stores answers.

## Concepts

Memoization
(Tabulation)

## Interview Questions

Q1: What is Dynamic Programming?

Answer:
An optimization technique that stores previously computed results.

Q2: When to use DP?

Answer:
When a problem has:
1. Overlapping Subproblems
2. Optimal Substructure

Q3: Difference between Recursion and DP?

Recursion:
Repeats calculations.

DP:
Stores results and avoids repetition.

## Example

def fib(n):
    dp = [0]*(n+1)

    dp[1] = 1

    for i in range(2,n+1):
        dp[i] = dp[i-1] + dp[i-2]

    return dp[n]
