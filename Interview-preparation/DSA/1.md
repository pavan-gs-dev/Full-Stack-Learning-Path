# 1. Arrays

## Definition
An Array is a collection of elements stored in contiguous memory locations.

Example:

[10, 20, 30, 40, 50]

Index:
0 1 2 3 4

Access:

Search = O(n)
Insert = O(n)
Delete = O(n)

## Interview Questions

An Array is a linear data structure that stores elements in contiguous memory locations.

Array:
- Fixed size
- Fast access

Linked List:
- Dynamic size
- Slow access

# 10. Dynamic Programming (DP)

## Definition

DP solves problems by storing previously calculated results.

Main Idea:

Don't calculate same thing again and again.

## Example

Fibonacci

Bad:

fib(5)
calls fib(4)
calls fib(3)
again calls fib(3)

Repeated calculations.

DP stores answers.

## Concepts

Memoization
(Tabulation)

## Interview Questions

Q1: What is Dynamic Programming?

Answer:
An optimization technique that stores previously computed results.

Q2: When to use DP?

Answer:
When a problem has:
1. Overlapping Subproblems
2. Optimal Substructure

Q3: Difference between Recursion and DP?

Recursion:
Repeats calculations.

DP:
Stores results and avoids repetition.

## Example

def fib(n):
    dp = [0]*(n+1)

    dp[1] = 1

    for i in range(2,n+1):
        dp[i] = dp[i-1] + dp[i-2]

    return dp[n]


# 1. Arrays

## Definition

An Array is a linear data structure that stores multiple elements of the same type in contiguous memory locations.

Example:

arr = [10, 20, 30, 40, 50]

Index:
0   1   2   3   4

Value:
10  20  30  40  50

---

## Why Do We Use Indexes?

An index is the position number of an element inside the array.

Example:

arr = [10,20,30,40,50]

arr[0] = 10
arr[1] = 20
arr[2] = 30

Without indexes, the computer would need to search element by element.

Using an index allows direct access to any element.

Example:

arr[3]

Output:
40

The computer can directly jump to the memory location of index 3.

---

## Why Is Array Access O(1)?

Suppose:

arr = [10,20,30,40,50]

Base Address = 1000
Size of Integer = 4 bytes

Formula:

Address = Base + (Index × Size)

For arr[3]:

Address = 1000 + (3 × 4)
Address = 1012

The computer directly calculates the address.

No searching is required.

Therefore:

Access = O(1)

O(1) means Constant Time.

Whether the array has:

5 elements
500 elements
500000 elements

Accessing arr[3] takes almost the same time.

---

## What Is Time Complexity?

Time Complexity tells us how the execution time grows when input size increases.

Input Size = n

Example:

Array:
[10,20,30,40,50]

Find 50

Check:
10
20
30
40
50

5 comparisons

If array has 1000 elements:

May require 1000 comparisons.

Time grows with input size.

This is O(n).

---

## Common Time Complexities

O(1)  = Constant Time
O(log n) = Logarithmic Time
O(n) = Linear Time
O(n log n) = Efficient Sorting
O(n²) = Nested Loops

---

## Array Time Complexities

Access by Index = O(1)

Search = O(n)

Insert at End = O(1)
(average case)

Insert at Beginning = O(n)

Delete = O(n)

---

## Example

arr = [10,20,30,40,50]

print(arr[2])

Output:
30

Time Complexity:
O(1)

Because we directly access index 2.
