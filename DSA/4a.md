# Introduction

A Stack mainly performs two operations:

1. Push
2. Pop

Think about a stack of books.

You can only:

* Put a new book on the top.
* Remove the top book.

You cannot insert or remove a book from the middle.

This is the basic rule of a Stack.

---

# Stack Before Any Operation

Initially, the stack is empty.

```text
Top

NULL
```

There are no elements.

---

# Operation 1 - Push

## What is Push?

Push means:

> Add a new element to the **top** of the stack.

It is called **Push** because we are pushing a new item onto the top.

---

# Real-Life Example

Imagine an empty plate stand.

Initially:

```text
Top

Empty
```

Place one plate.

```text
Top

Plate 1
```

Place another plate.

```text
Top

Plate 2

Plate 1
```

Place another plate.

```text
Top

Plate 3

Plate 2

Plate 1
```

Every new plate goes on the top.

Exactly the same happens in a Stack.

# Push Using Python List

Python does not have a built-in Stack class.

Instead, we commonly use a List.

```python
stack = []
```

Initially:

```text
[]
```

Push 10

```python
stack.append(10)
```

Stack becomes

```text
[10]
```

Push 20

```python
stack.append(20)
```

Stack becomes

```text
[10,20]
```

Push 30

```python
stack.append(30)
```

Stack becomes

```text
[10,20,30]
```

Remember:

Although Python prints the list from left to right, the **last element** is considered the **Top** of the Stack.

Visualization

```text
Top

30

20

10
```

# Time Complexity of Push

Push always inserts at the top.

No searching is required.

Therefore,

```text
Time Complexity = O(1)
```

Constant Time.

---

# Space Complexity of Push

Each Push adds one element.

```text
O(1)
```

per operation.

# Operation 2 - Pop

## What is Pop?

Pop means:

> Remove the top element from the stack.

Remember:

The last inserted element is always removed first.

---

# Real-Life Example

Stack of books

Before

```text
Top

Python

Java

C++
```

Pop

Python is removed.

Remaining

```text
Top

Java

C++
```

You never remove the middle book first.

---

# Pop Using Python

Python uses

```python
pop()
```

Example

```python
stack = [10,20,30]

stack.pop()

print(stack)
```

Output

```text
[10,20]
```

---

# Complete Program

```python
stack = []

stack.append(10)
stack.append(20)
stack.append(30)

print("Before Pop:", stack)

removed = stack.pop()

print("Removed:", removed)

print("After Pop:", stack)
```

Output

```text
Before Pop: [10, 20, 30]

Removed: 30

After Pop: [10, 20]
```

---

# What Happens if Stack is Empty?

Example

```python
stack = []

stack.pop()
```

Output

```text
IndexError: pop from empty list
```

Why?

Because there is nothing to remove.

Always check before calling `pop()`.

---

# Time Complexity of Pop

Pop removes the top element.

No searching required.

Time Complexity

```text
O(1)
```

---

# Space Complexity of Pop

No extra memory required.

```text
O(1)
```

---

# Push vs Pop

| Push                 | Pop                     |
| -------------------- | ----------------------- |
| Adds a new element   | Removes the top element |
| Uses append()        | Uses pop()              |
| Stack size increases | Stack size decreases    |
| Top changes          | Top changes             |

# Common Interview Questions

### Q1. What is Push?

Adding an element to the top of the stack.

---

### Q2. What is Pop?

Removing the top element from the stack.

---

### Q3. Which Python function performs Push?

```python
append()
```

---

### Q4. Which Python function performs Pop?

```python
pop()
```

---

### Q5. What happens when `pop()` is called on an empty stack?

Python raises:

```text
IndexError
```

---

### Q6. Why is Push O(1)?

Because the element is always inserted at the top without traversing the stack.

---

### Q7. Why is Pop O(1)?

Because only the top element is removed.
