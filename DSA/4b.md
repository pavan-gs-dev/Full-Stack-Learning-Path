# Operation 3 - Peek (Top)

## What is Peek?

Peek means:

> View the top element **without removing it**.

Think of it like looking at the top book in a stack.

You only look.

You do **not** remove it.

---

# Real-Life Example

Books:

```text
Top

Python

Java

C++
```

If someone asks,

"What is the top book?"

Answer:

```text
Python
```

But the books remain exactly the same.

Nothing is removed.

---

# Stack Visualization

Current Stack

```text
Top
 ↓

30

20

10
```

Peek returns

```text
30
```

Stack after Peek

```text
Top
 ↓

30

20

10
```

Notice:

Nothing changed.

---

# Peek Using Python

Python List

```python
stack = [10,20,30]
```

Top element is always the last element.

```python
print(stack[-1])
```

Output

```text
30
```

---

# Why -1?

Python supports negative indexing.

Example

```python
stack = [10,20,30]
```

| Index | Value |
| ----- | ----- |
| 0     | 10    |
| 1     | 20    |
| 2     | 30    |
| -1    | 30    |
| -2    | 20    |
| -3    | 10    |

So

```python
stack[-1]
```

always returns the top element.

---

# Complete Program

```python
stack = []

stack.append(10)
stack.append(20)
stack.append(30)

print("Top Element:", stack[-1])

print("Stack:", stack)
```

Output

```text
Top Element: 30

Stack: [10,20,30]
```

Notice:

30 was not removed.

---

# Dry Run

Initial Stack

```text
[10,20,30]
```

Peek

```python
stack[-1]
```

Returns

```text
30
```

Stack after Peek

```text
[10,20,30]
```

No changes.

---

# Time Complexity

Peek only reads one element.

No searching.

```text
O(1)
```

Space Complexity

```text
O(1)
```

# Operation 4 - isEmpty()

## What is isEmpty()?

This operation checks whether the Stack contains any elements.

Returns

```text
True
```

if Stack is empty.

Returns

```text
False
```

if Stack contains elements.

---

# Why is isEmpty() Important?

Imagine calling

```python
stack.pop()
```

on an empty stack.

Python throws

```text
IndexError
```

So before Pop, interviewers often ask you to check

```text
isEmpty()
```

---

# Using Python

```python
stack = []
```

Check

```python
if len(stack) == 0:
    print("Stack is Empty")
```

Output

```text
Stack is Empty
```

---

# Another Method

Python allows

```python
if not stack:
    print("Empty")
```

Explanation

Empty List

```python
[]
```

is treated as

```text
False
```

So

```python
not stack
```

becomes

```text
True
```

---

# Dry Run

Initially

```python
stack = []
```

Length

```python
len(stack)
```

returns

```text
0
```

Therefore

```python
len(stack)==0
```

returns

```text
True
```

---

# Time Complexity

Only checks the length.

```text
O(1)
```

---

# Operation 5 - Size

## What is Size?

Size tells us

> How many elements are currently inside the Stack.

---

# Example

```python
stack = [10,20,30]
```

Size

```python
print(len(stack))
```

Output

```text
3
```

---

# Dry Run

Stack

```text
10

20

30
```

Python counts

```text
1

2

3
```

Returns

```text
3
```

---

# Time Complexity

Python stores the length internally.

So

```python
len(stack)
```

is

```text
O(1)
```

# Operation 6 - Traverse

## What is Traversing?

Traverse means

> Visit every element one by one.

Suppose Stack

```text
Top

40

30

20

10
```

Traversing means

```text
40

30

20

10
```

---

# Python Program

```python
stack = [10,20,30,40]

for item in reversed(stack):
    print(item)
```

Output

```text
40

30

20

10
```

---

# Why reversed()?

Python stores

```text
10

20

30

40
```

inside the list.

But Stack works from

```text
40

30

20

10
```

So we reverse the display.

---

# Dry Run

Stack

```text
[10,20,30,40]
```

Traversal

```text
40

30

20

10
```

---

# Time Complexity

Visits every element.

```text
O(n)
```

---

# Complete Stack Example

```python
stack = []

stack.append(10)
stack.append(20)
stack.append(30)

print("Current Stack:", stack)

print("Top:", stack[-1])

print("Size:", len(stack))

print("Is Empty:", len(stack)==0)

print("Traverse")

for item in reversed(stack):
    print(item)

removed = stack.pop()

print("Removed:", removed)

print("After Pop:", stack)
```

Output

```text
Current Stack: [10,20,30]

Top: 30

Size: 3

Is Empty: False

Traverse

30
20
10

Removed: 30

After Pop: [10,20]
```

---

# Time Complexity Table

| Operation | Complexity |
| --------- | ---------- |

| Operation | Complexity       |

# Common Mistakes

## Mistake 1

```python
stack.pop()
```

on an empty stack.

Wrong

Always check

```python
if stack:
```

before calling `pop()`.

---

## Mistake 2

Thinking

```python
stack[0]
```

is Top.

Wrong.

Top is

```python
stack[-1]
```

---

## Mistake 3

Printing Stack from first element.

Correct traversal

```python
reversed(stack)
```

---

## Mistake 4

Confusing Stack with Queue.

Remember

Stack

```text
LIFO
```

Queue

```text
FIFO
```
