# Chapter 1 - What is a Stack?

## Introduction

A **Stack** is a **Linear Data Structure** that follows the **LIFO (Last In, First Out)** principle.

This means:

> The last element inserted into the stack is the first element removed from the stack.

Think of a stack as a pile of books.

You always place a new book on the **top**, and when you remove a book, you also remove it from the **top**.

You **cannot remove a book from the middle or the bottom** without removing the books above it.

---

# Definition

A Stack is a linear data structure in which insertion and deletion happen only from one end called the **Top**.

The two most important operations are:

* Push → Insert an element.
* Pop → Remove the top element.

---

# Why Do We Need a Stack?

Many computer programs need to remember the **most recent item**.

Examples:

* Browser Back button
* Undo in Microsoft Word
* Redo operation
* Function calls in Python
* Calculator expression evaluation
* Parentheses matching
* DFS (Depth-First Search)

All these problems naturally follow the **Last In, First Out** rule.

# Real-Life Example 1 - Stack of Plates

Imagine a restaurant.

A waiter stacks plates like this:

```text
     _______
    | Plate5 |
    |-------|
    | Plate4 |
    |-------|
    | Plate3 |
    |-------|
    | Plate2 |
    |-------|
    | Plate1 |
    |_______|
```

A new plate is always placed on the top.

When someone needs a plate, the waiter removes the top plate first.

He never removes Plate 2 before Plate 5.

This is exactly how a Stack works.

# LIFO (Last In First Out)

Stack follows the **LIFO** rule.

Meaning:

The element inserted last is removed first.

Example:

Push:

```text
Push 10

Push 20

Push 30
```

Stack becomes:

```text
Top

30

20

10
```

Now Pop:

```text
30 removed
```

Remaining:

```text
Top

20

10
```

Pop again:

```text
20 removed
```

Remaining:

```text
Top

10
```

Notice:

30 entered last.

30 left first.

This is called **Last In First Out (LIFO)**.

# Memory Representation

Suppose we Push four numbers.

```text
Push 10

Push 20

Push 30

Push 40
```

Memory:

```text
Top
 ↓
+------+
|  40  |
+------+
|  30  |
+------+
|  20  |
+------+
|  10  |
+------+
```

Top always points to the newest element.

# Stack Terminology

## Top

The top is the last inserted element.

Example:

```text
Top
 ↓
40
30
20
10
```

Top = 40

---

## Bottom

The first inserted element.

```text
10
```

Bottom rarely changes.

---

## Push

Adding a new element on top.

Example:

Before:

```text
30

20

10
```

Push 40

After:

```text
40

30

20

10
```

---

## Pop

Removing the top element.

Before:

```text
40

30

20

10
```

Pop

After:

```text
30

20

10
```

---

## Peek (Top)

Peek means seeing the top element without removing it.

Example:

```text
40

30

20
```

Peek returns:

```text
40
```

Stack remains unchanged.

# Stack vs Array

| Array            | Stack                |
| ---------------- | -------------------- |
| Access any index | Access only the top  |
| Insert anywhere  | Insert only at top   |
| Delete anywhere  | Delete only from top |
| Uses indexes     | Uses Top pointer     |
| Flexible access  | Restricted access    |

Think of a Stack as a special way of using an array.

# Advantages of Stack

✅ Easy to implement

Example:


# Time Complexity Overview

| Operation | Complexity |
| --------- | ---------- |
| Push      | O(1)       |
| Pop       | O(1)       |
| Peek      | O(1)       |
| Search    | O(n)       |

Why?

Because Push, Pop and Peek always happen at the top.

Searching may require checking every element.
