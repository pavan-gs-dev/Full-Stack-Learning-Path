# Introduction

Before solving DSA problems, we must understand:

1. Time Complexity
2. Space Complexity

These help us measure how efficient an algorithm is.

---

# What is Time Complexity?

Time Complexity tells us how the running time of an algorithm grows when the input size increases.

It does NOT measure actual time in seconds.

Instead, it measures the number of operations performed.

Example:

```python
arr = [10,20,30,40,50]

for num in arr:
    print(num)
```

If array contains:

```text
5 elements
```

Loop runs:

```text
5 times
```

If array contains:

```text
1000 elements
```

Loop runs:

```text
1000 times
```

Time Complexity:

```text
O(n)
```

---

# Why Time Complexity is Important?

Suppose two developers solve the same problem.

Solution A

```python
for i in range(n):
    print(i)
```

Solution B

```python
for i in range(n):
    for j in range(n):
        print(i,j)
```

For:

```text
n = 1000
```

Solution A:

```text
1000 operations
```

Solution B:

```text
1,000,000 operations
```

Clearly Solution A is faster.

---

# Big O Notation

We represent Time Complexity using Big O notation.

Examples:

```text
O(1)
O(log n)
O(n)
O(n log n)
O(n²)
O(n³)
O(2ⁿ)
O(n!)
```

---

# O(1) - Constant Time

Execution time never changes.

Example:

```python
arr = [10,20,30,40,50]

print(arr[2])
```

No matter whether array contains:

```text
5 elements
500 elements
5000 elements
```

Python directly accesses index 2.

Time Complexity:

```text
O(1)
```

Best possible complexity.

---

# O(log n) - Logarithmic Time

Commonly seen in Binary Search.

Example:

```text
Array:
[10,20,30,40,50,60,70,80]
```

Find:

```text
70
```

Step 1:

```text
Check middle = 40
```

Step 2:

```text
Discard first half
```

Step 3:

```text
Check middle again
```

Found.

Instead of checking all elements, we keep dividing by 2.

Time Complexity:

```text
O(log n)
```

Very efficient.

---

# O(n) - Linear Time

Work grows linearly with input size.

Example:

```python
arr = [10,20,30,40,50]

for num in arr:
    print(num)
```

Operations:

```text
5 elements = 5 operations

100 elements = 100 operations

1000 elements = 1000 operations
```

Time Complexity:

```text
O(n)
```

---

# O(n²) - Quadratic Time

Usually occurs with nested loops.

Example:

```python
for i in range(n):
    for j in range(n):
        print(i,j)
```

Operations:

```text
n × n
```

For:

```text
1000
```

Operations:

```text
1000 × 1000

= 1,000,000
```

Time Complexity:

```text
O(n²)
```

Expensive for large inputs.

---

# O(n³) - Cubic Time

Three nested loops.

Example:

```python
for i in range(n):
    for j in range(n):
        for k in range(n):
            print(i,j,k)
```

Complexity:

```text
O(n³)
```

Avoid whenever possible.

---

# O(n log n)

Used in efficient sorting algorithms.

Examples:

* Merge Sort
* Heap Sort
* Quick Sort (Average Case)

Complexity:

```text
O(n log n)
```

Very common in interviews.

---

# O(2ⁿ)

Exponential complexity.

Example:

```python
def fibonacci(n):
    if n <= 1:
        return n

    return fibonacci(n-1) + fibonacci(n-2)
```

Complexity:

```text
O(2ⁿ)
```

Very slow.

---

# O(n!)

Factorial complexity.

Example:

Generating all permutations.

```text
ABC

ACB

BAC

BCA

CAB

CBA
```

Complexity:

```text
O(n!)
```

Worst among common complexities.

---

# Complexity Ranking

Best to Worst

```text
O(1)

↓

O(log n)

↓

O(n)

↓

O(n log n)

↓

O(n²)

↓

O(n³)

↓

O(2ⁿ)

↓

O(n!)
```

# Time Complexity of Array Operations

| Operation    | Complexity |
| ------------ | ---------- |
| Access       | O(1)       |
| Update       | O(1)       |
| Traverse     | O(n)       |
| Search       | O(n)       |
| Insert       | O(n)       |
| Delete       | O(n)       |
| Reverse      | O(n)       |
| Find Maximum | O(n)       |
| Find Minimum | O(n)       |

---

# How to Calculate Time Complexity

## Example 1

```python
for i in range(n):
    print(i)
```

Runs n times.

Answer:

```text
O(n)
```

---

## Example 2

```python
for i in range(n):
    for j in range(n):
        print(i,j)
```

Runs:

```text
n × n
```

Answer:

```text
O(n²)
```

---

## Example 3

```python
print(arr[5])
```

Runs once.

Answer:

```text
O(1)
```

---

## Example 4

```python
while n > 1:
    n = n // 2
```

Input keeps dividing by 2.

Answer:

```text
O(log n)
```

# What is Space Complexity?

Space Complexity measures the amount of memory used by an algorithm.

Example:

```python
arr = [10,20,30,40]
```

Memory is required to store:

```text
10
20
30
40
```

Space Complexity:

```text
O(n)
```

---

# Space Complexity Examples

## Example 1

```python
a = 10
b = 20
```

Only two variables.

Space Complexity:

```text
O(1)
```

---

## Example 2

```python
arr = [1,2,3,4,5]
```

Memory grows with input.

Space Complexity:

```text
O(n)
```

---

## Example 3

```python
matrix = [
 [1,2,3],
 [4,5,6],
 [7,8,9]
]
```

Rows × Columns

Space Complexity:

```text
O(n²)
```
